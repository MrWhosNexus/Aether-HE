`~/Projects/aula-win60-app` has TWO front-ends:
- `app_web.py` — **the one the user wants**: renders the exact Claude Design ("AETHER HE // Keyboard Hub") from `ui/index.html` (a self-contained React/Tailwind standalone exported from claude.ai/design) in a GTK/WebKit window via **pywebview**. Run with: `venv-web/bin/python app_web.py`.
- `main.py` — an earlier CustomTkinter reimplementation (tradeos/AETHER-styled). Superseded by the webview; keep only if asked.

**Env:** `venv-web` was created with `python -m venv --system-site-packages` so it can import the system `gi` + `WebKit2 4.1` (the plain `.venv` cannot). It also has pip `pywebview`, `hidapi`, `evdev`. System python3 (3.14) has gi+WebKit2 but no `hid`.

**HID bridge (REWRITTEN 2026-05-24 — bridge is now baked into React, NOT scraped):** WebKit has NO WebHID. `app_web.py` exposes a Python `Api` as `window.pywebview.api`. The app loads `ui/index_runtime.html` (falls back to `ui/index.html` if absent). `index_runtime.html` is a GENERATED build: run `python ui/runtime_src/build_runtime.py` to regenerate it. That builder Babel-precompiles the JSX (using the bundle's own `vendor/babel.js`, build-time only) and concatenates React+ReactDOM+Tailwind + the design modules. The design modules were decoded out of `ui/index.html`'s `__bundler/manifest` (gzip+base64) into `ui/runtime_src/`: `vendor/` = unmodified (react, react-dom, tailwind, babel, icons.jsx, theme.jsx); `src/` = OUR edited modules (`keyboard.jsx`, `sections.jsx`, `app.jsx`). **Every control calls `window.pywebview.api.*` directly with structured values — there is NO DOM/CSS scraping anymore.** `app.jsx` holds all state + the bridge effects/handlers (connect/disconnect, auto-connect persisted in localStorage, lighting `set_light`, custom per-key `set_custom_colors`, actuation `set_trigger_codes`, dead band, switch, poll, SOCD `set_socd`, calibration, travel-test `open_analog_codes`+`read_live` poll). `keyboard.jsx`: removed the simulated WASD press loop — `KeyboardPanel` now takes a `liveDepths` prop fed by real `read_live()`. `app_web.py`'s old giant `BRIDGE_JS` is GONE; it now injects only a tiny `NATIVE_JS` (physical-key-event suppression + the PAD gamepad-capture toggle). Tab switches no longer re-send packets (section is intentionally excluded from the lighting/actuation effect deps). TWO-COLOR EFFECTS: `build_light` carries a foreground (bytes 8-10) AND a background (bytes 11-13), and most effects use both — confirmed in `decoded/deobfuscated.js` ~L2155-2265, which shows the bg-color picker for modes {1,2,3,4,6,7,8,9(Starlight),11,14,15,16}; only Static(0)/Custom(10)/Octane(12) are fg-only. `app.jsx` maps fg=`colors[0]`, bg=the Background-color picker. FIRMWARE CANNOT BE MODIFIED (closed MCU, no source/SDK, reflash bricks) so to get 2+ colors on an animated effect the app drives a HOST per-key engine: `effects.py` `PerKeyEffectEngine` streams per-key RGB (~18fps via `protocol.build_custom_light`, cmd 9) to animate twinkle/wave/breath/ripple with the full palette. Api: `start_multicolor(pattern, colors, bg, brightness, speed)` (sets Custom mode then runs engine) + `stop_multicolor`; `_send_frame` writes the frame packets. `app.jsx` lighting effect: if `colors.length>=2` and power on and not Full-RGB/Static → `start_multicolor`, else `stop_multicolor` + firmware `set_light`. Full-RGB toggle (byte 15 `full_color`) = rainbow spectrum. Direction control (→←↑↓) + Background-brightness slider (pre-scales bg RGB). EFFECT ZONES: engine is `Zone`-based — `start_zones([{codes,mode,colors,bg,speed,brightness,direction}])` composites multiple key-group effects per frame (Custom mode `ZonesPanel` UI: select keys → + Zone → pick effect/colors/speed). Distinct generators: twinkle/wave/striation(hard bands)/radar(rotating)/ripple/cross/fireworks/aurora/breath/static. Per-key custom color order is RGB (matches driver, verified pure R/G/B on hardware — no hue swap). ALL animated effects (every non-Static, non-FullRGB pattern, any color count) now route through the host engine so behavior is consistent + the in-app keyboard can mirror it; only Static + Full-RGB use firmware `set_light`. In-app keyboard mirror: `Api.get_light_frame()` returns `{code:[r,g,b]}` of `fx.last_frame`; `app.jsx` polls it (~15fps, Lighting tab) → `ledMapLive` → KeyboardPanel. GAMMA(2.2) applied to PALETTE only (Zone init), bg stays linear (fixes orange→yellow without crushing bg). `reactive` = press-driven (`fx.get_depths` fed from a LiveReader, `_ensure_reactive`). Latency: `_send_frame` diffs the 8 custom-light pages and only re-sends changed ones. Selected keys render a white ring in lighting mode (`mode==='lighting'`). Drag-select on the board (mousedown+mouseenter). Effect-colors palette hidden in Custom (colors come from zones). GAMEPAD: host uinput gamepad (`gamepad.py`) needs `/dev/uinput` writable — install `99-uinput.rules`, `usermod -aG input $USER`, set node `chgrp input /dev/uinput && chmod 660`; uinput is BUILT-IN on CachyOS kernel (no modprobe). Group change needs re-login. See [[aula-win60-protocol-capture]] and [[aula-win60-lighting-modes]].

**Full protocol decoded** from the official driver: fetched `hed.aulacn.com` JS, ran `npx webcrack` → `driver_src/dec_agreement/deobfuscated.js` is the HID protocol module. Extracted verbatim into `protocol.py` — all Report ID 1, body[0]=cmd: `build_light` (cmd 7; [5]mode [6]brightness [7]speed [8-10]**fg** [11-13]**bg** [14]dir [16]power — the old `ff/55` guess was actually background color), `build_trigger` (cmd 33 actuation: key bitmask + travel/interval1/interval2), `build_poll_rate` (cmd 33/9), `build_open_trigger_test` (cmd 33/2 = enable analog stream), `build_prcs` (cmd 36 = SOCD: model+key1/key2 hidCode), `build_gamepad_mode` (cmd 20/3), `build_win_lock` (cmd 20). Key layout from `config/keys/SI2825HEARGB.json` (bundled `ui/keymap.json`): index/hidCode/x/y per key — W=index46 (row2,col2), A=68, S=69, D=70, Esc=22; device index = row*22+col. `device_state.py` = KeyMap + LiveReader (enables travel-test, parses cmd33/sub5: idx=[7]*22+[8], depth=([9]|[10]<<8)/100 mm). Reads VALIDATED on hardware (info handshake, heartbeat, travel-test ACK). `app_web.py` now wires lighting+polling+actuation, and the bridge kills the prototype's *simulated* travel and glows keys by real `api.read_live()` depth.

`effects.py` = host-driven animated effects (the user did NOT want these — superseded by firmware modes via build_light). Streamed via global color if ever used.
